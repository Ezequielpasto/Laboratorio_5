Motores de bases de datos más utilizados:

MySQL: Es una base de datos relacional muy popular y facil de usar. Organiza la información en tablas y utiliza SQL para hacer consultas, es utilizado mucho en páginas web, aplicaciones y sistemas pequeños o medianos.

PostgreSQL: Es una base de datos relacional avanzada y muy potente, posee mas funciones que MySQL y muy valorada por su seguridad, esabilidad y capacidad para manejar proyectos grandes.

MariaDB: Es una alternativa open souce que fue creada a partir de MySQL. Mantiene mucha compatibilidad con MySQL, pero agrega mejoras de rendimiento y nuevas funciones.

MongoDB: Es una base de datos NoSQL que guarda la información en documentos parecidos a JSON en vez de tablas tradicionales, es flexible y utilizada en aplicaciones modernas.

Redis: Es una base de datos extremadamente rápida porque trabaja principalmente en memoria RAM, utilizada para caché, sesiones de usuarios, chats en tiempo real y sistemas que requieran de mucha velocidad.

SQLite: Es una base de datos ligera que funciona dentro de un solo archivo, sin falta de instalar un servidor y se utiliza en aplicaciones moviles y en programas chicos.

Apache Cassandra: Es una base de datos NoSQL diseñada para manejar enormes cantidades de datos distribuidos en muchos servidores, pensada para utilizar en sistemas muy grandes.

Oracle Database: Es una base de datos empresarial muy potente que se destaca por su seguridad, rendimiento y herramientas avanzada para mejorar sistemas críticos.

Microsoft SQL Server: Es la base de datos desarrollada por Microsoft que usa SQL y se integra muy bien con tecnologias como .NET y Windows Serve, siendo muy comun en entornos empresariales.

Diferencia entre VM y contenedor: Una máquina virtual o VM virtualiza todo un sistema operativo incluyendo su propio Kernel, lo que consume más recursos y tarda mas en iniciar. En cambio un contenedor comparte kernel del sistema operativo alfitrion y solo incluye la aplicacion y sus dependencias.

¿Qué proceso mantiene vivo el contenedor?
Un contenedor se mantiene vivo mientras el proceso principal se ejecuta siga funcionando. Si ese proceso termina, el contenedor automaticamente se detiene.

Ventajas de Docker: Docker permite crear y ejecutar contenedores de manera sencilla y portable. Sus principales ventajas son el bajo consumo de recursos, la rapidez al iniciar aplicaciones, su facilidad para mover proyectos entre dificiles máquinas, la consistencia del entorno de desarrollo y la posibilidad de aislar aplicaciones y dependencias sin conflictos.
